## Section 2: Scoring Methodology

### The 10 Dimensions

Each service is scored 0-10 across ten dimensions. The rubric for each dimension is:

#### 1. UPTIME (Weight: x2.0)

| Score | Criteria |
|-------|----------|
| **10** | > 99.9% uptime (30-day rolling), process uptime > 30 days |
| **7**  | > 99% uptime, process uptime > 7 days |
| **4**  | > 95% uptime, recent restarts observed |
| **0**  | Frequent crashes, restart loops, unmonitored |

**Data source:** Uptime Kuma logs, PM2 `status` output, `/proc/uptime` per server.  
**Validation command:** `ssh <host> 'pm2 jlist' | jq '[.[].pm2_env.pm_uptime]'`

#### 2. LOGS (Weight: x1.0)

| Score | Criteria |
|-------|----------|
| **10** | Structured logging (JSON), log aggregation via Loki, no ERROR patterns in 24h |
| **7**  | Structured logging, logs written to file, no critical errors |
| **4**  | Unstructured logs, some error patterns present |
| **0**  | No logging, log file full, errors flooding |

**Data source:** Loki `rate({job=~".+"} |= "ERROR" [24h])`, PM2 log files.  
**Validation command:** `logcli query 'sum(count_over_time({job=~".+"} |~ "ERROR|FATAL" [24h]))'`

#### 3. HEALTH ROUTE (Weight: x1.5)

| Score | Criteria |
|-------|----------|
| **10** | Dedicated `/health` endpoint returning JSON status with dependency checks (DB, Redis reachable) |
| **7**  | Health endpoint exists, returns HTTP 200 |
| **4**  | Health endpoint exists but unreliable (sporadic failures, timeout) |
| **0**  | No health endpoint |

**Data source:** Direct HTTP probe.  
**Validation command:** `curl -s -o /dev/null -w "%{http_code}" http://<host>:<port>/health`

#### 4. DEPENDENCIES (Weight: x1.5)

| Score | Criteria |
|-------|----------|
| **10** | All deps monitored, circuit breakers configured (opossum/polly), graceful degradation paths |
| **7**  | All deps documented, connection pooling configured, timeouts set |
| **4**  | Some dependency issues, timeouts not configured, no connection pooling |
| **0**  | Hard dependencies with no error handling, cascade failure risk |

**Data source:** Code review, runtime dependency graph, connection pool metrics.  
**Validation command:** Review source for circuit breaker imports; check env vars for timeout settings.

#### 5. OBSERVABILITY (Weight: x1.5)

| Score | Criteria |
|-------|----------|
| **10** | Prometheus metrics exported, dedicated Grafana dashboard, Loki log aggregation, alert rules configured |
| **7**  | Metrics exported (prom-client or equivalent), basic dashboard exists |
| **4**  | Some metrics exposed, no dedicated dashboard |
| **0**  | No observability — no metrics, no logs, no alerts |

**Data source:** Prometheus `up` metric, Grafana dashboard inventory.  
**Validation command:** `curl -s http://<host>:9090/api/v1/targets | jq '.data.activeTargets | length'`

#### 6. ROLLBACK (Weight: x2.0)

| Score | Criteria |
|-------|----------|
| **10** | Automated rollback, tested in last 30 days, completes in < 5 minutes |
| **7**  | Documented rollback procedure, manual but verified (someone has executed it) |
| **4**  | Rollback plan exists but untested |
| **0**  | No rollback plan |

**Data source:** Deployment scripts, CI/CD pipeline config, incident runbooks.  
**Validation script:** `/root/scripts/revenue-rollback-checklist.sh`

#### 7. BACKUP (Weight: x2.0)

| Score | Criteria |
|-------|----------|
| **10** | Automated daily backups, off-site copy (different provider/region), restore tested in last 30 days |
| **7**  | Automated daily backups, off-site copy, restore not tested recently |
| **4**  | Manual backups, no restore test |
| **0**  | No backups configured |

**Data source:** Cron jobs, backup logs, MinIO bucket listings, restore test reports.  
**Validation command:** `ssh coredb 'ls -la /var/backups/ && pg_dump --schema-only > /dev/null 2>&1 && echo "pg_dump OK"'`

#### 8. SECURITY (Weight: x1.5)

| Score | Criteria |
|-------|----------|
| **10** | All security headers (HSTS, CSP, X-Frame-Options), HTTPS only, no CVEs > 7 (HIGH/CRITICAL), auth required for sensitive endpoints |
| **7**  | HTTPS everywhere, auth implemented, recent (< 90 days) security audit |
| **4**  | HTTPS, basic auth, some security headers |
| **0**  | HTTP only, no auth, known unpatched vulnerabilities |

**Data source:** OWASP ZAP scan, `nmap` port scan, CVE database.  
**Validation command:** `curl -sI https://<host> | grep -E 'strict-transport-security|x-frame|x-content'`

#### 9. MIGRATION READINESS (Weight: x1.0)

| Score | Criteria |
|-------|----------|
| **10** | All 9 migration gates documented and pre-verified |
| **7**  | Migration plan documented, gates defined with acceptance criteria |
| **4**  | Migration planned at a high level but not documented in detail |
| **0**  | No migration plan exists |

The 9 Migration Gates are:
1. DNS TTL lowered to 60s (verified)
2. SSL certificates pre-provisioned on target
3. Database dump/restore tested end-to-end
4. Redis snapshot and AOF verified
5. MinIO bucket mirror confirmed
6. Environment variables synced to target
7. Smoke test suite passes on target
8. Rollback procedure documented and timed
9. Stakeholder communication plan approved

**Validation script:** `/root/scripts/revenue-rollback-checklist.sh`

#### 10. REVENUE IMPACT (Weight: x1.0)

| Score | Criteria |
|-------|----------|
| **10** | Non-revenue internal tool (low impact if down for hours) |
| **7**  | Internal productivity tool (slows team if down but no customer impact) |
| **4**  | Customer-facing but not directly revenue-generating |
| **0**  | Direct revenue impact if down (main site, CRM, payment processing) |

> **Note on Revenue Impact scoring:** This dimension is scored **inversely** — a higher score means *lower* revenue risk. A service that generates direct revenue scores 0 (maximum penalty applied when the service is unhealthy). This ensures revenue-critical services face the highest bar for production readiness.

### Weighting Rationale

| Dimension | Weight | Rationale |
|-----------|--------|-----------|
| **Uptime** | x2.0 | Directly measures reliability — the most visible indicator of production readiness. Poor uptime undermines everything else. |
| **Rollback** | x2.0 | The single most important safety mechanism during migration. Without reliable rollback, every deployment is a one-way door. |
| **Backup** | x2.0 | Data loss is irreversible. Backups are the last line of defense and must be weighted accordingly. |
| **Health Route** | x1.5 | Load balancer and orchestration dependency — without it, there is no auto-recovery, no graceful restarts. |
| **Dependencies** | x1.5 | Dependency failures are the #1 cause of cascading outages in microservice architectures. |
| **Observability** | x1.5 | Without observability you cannot detect, diagnose, or improve anything else. "You can't fix what you can't see." |
| **Security** | x1.5 | Security compromises cascade into downtime, data loss, and revenue loss. A breach is a forced outage. |
| **Logs** | x1.0 | Foundational but secondary to aggregating those logs into observability (Loki/Grafana). |
| **Migration Readiness** | x1.0 | Important contextually (we are actively migrating) but temporary — weight will drop post-migration. |
| **Revenue Impact** | x1.0 | Contextualizer — amplifies the effective importance of all other scores for revenue-critical services. |

### Normalization to 0-100

```
MAX_WEIGHTED_POINTS = (10 x 2.0) + (10 x 1.0) + (10 x 1.5) + (10 x 1.5) + (10 x 1.5)
                    + (10 x 2.0) + (10 x 2.0) + (10 x 1.5) + (10 x 1.0) + (10 x 1.0)

                    = 20 + 10 + 15 + 15 + 15 + 20 + 20 + 15 + 10 + 10
                    = 150 points

FINAL_SCORE = (actual_weighted_points / 150) x 100
```

| Gate | Description | Validation Command | Status |
|------|-------------|--------------------|--------|
| **G1** | DNS TTL lowered to 60s on all domains | `dig +short fundsrecoverygroup.com && dig SOA fundsrecoverygroup.com` | [ ] |
| **G2** | SSL certificates pre-provisioned on target | `curl -sI https://<target-ip> | grep "200 OK"` | [ ] |
| **G3** | PostgreSQL dump/restore tested end-to-end | `pg_dump | pg_restore` on target, verify row counts match | [ ] |
| **G4** | Redis snapshot and AOF verified | `redis-cli SAVE`, copy RDB, restore, `DBSIZE` match | [ ] |
| **G5** | MinIO bucket mirror confirmed | `mc mirror --dry-run source/bucket target/bucket` | [ ] |
| **G6** | Environment variables synced to target | Diff `.env.production` between source and target | [ ] |
| **G7** | Smoke test suite passes on target | Run `npm run test:smoke` against target hosts | [ ] |
| **G8** | Rollback procedure documented and timed | Run rollback script, time it, document result | [ ] |
| **G9** | Stakeholder communication plan approved | Signed off by CTO, Product, and Support leads | [ ] |
